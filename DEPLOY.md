## Who Owns and Maintains the Data?

```
┌─────────────────────────────────────────────────────────────────┐
│                     DATA OWNERSHIP CHAIN                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  YOU (Kumar Swamy / bkumars22)                                  │
│  ├─ Own the GitHub repo (source code)                           │
│  ├─ Own the Render.com account (servers + database)             │
│  ├─ Set all passwords and API keys                              │
│  ├─ Legal Data Controller — responsible for all stored data     │
│  └─ Only you can directly access the database                   │
│                                                                 │
│  RENDER.COM (your chosen cloud host)                            │
│  ├─ Runs the Spring Boot server on their machines               │
│  ├─ Runs the PostgreSQL database on their machines              │
│  └─ Data physically stored in Singapore/Oregon datacenters      │
│                                                                 │
│  END USERS (teachers, parents)                                  │
│  ├─ Access ONLY through the React frontend UI                   │
│  ├─ API enforces role — parents see only their children         │
│  └─ Cannot touch database directly — ever                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**You are 100% in control.** Render cannot read your users' data.
End users cannot access the database. Only your Spring Boot API can.

## What Sensitive Data Is Stored?

| Data | Table | Who Can See It |
|------|-------|----------------|
| Student full name, age, grade | `STUDENT` | Teacher (own students), Parent (own child only), Admin |
| Parent/Teacher name + email | `ARIA_USER` | Admin only |
| Passwords | `ARIA_USER` | Stored as BCrypt hash — **never readable, not even by you** |
| Learning sessions | `LEARNING_SESSION` | Teacher (own students), Parent (own child) |
| Chat messages with AI | `SESSION_MESSAGE` | Teacher, Parent (own child) |
| Progress/scores | `STUDENT_PROGRESS` | Teacher, Parent (own child) |

**No payment data, no Aadhaar, no phone numbers are stored.**

## Step-by-Step: Deploy to Render.com (Free)

### Prerequisites
- GitHub account with the ARIA repo

1. Go to https://render.com → **Get Started for Free**
2. Sign up with your GitHub account (bkumars22)

### Step 2 — Deploy via Blueprint (One Click)

1. In Render dashboard → **New** → **Blueprint**
2. Connect to repo: `bkumars22/ARIA`
3. Render reads `render.yaml` automatically
4. It creates:
   - `aria-db` (PostgreSQL database — free)
   - `aria-backend` (Spring Boot API — free)
   - `aria-ai` (FastAPI AI service — free)

---

### Step 3 — Set Your Secret Keys

In Render dashboard → `aria-ai` service → **Environment**:

| Key | Value | Where to get |
|-----|-------|-------------|
| `ANTHROPIC_API_KEY` | `sk-ant-...` | https://console.anthropic.com → API Keys |

The `JWT_SECRET` is auto-generated by Render (random strong key).
The database URL/user/password are auto-wired from `aria-db`.

---

### Step 4 — Update Frontend to Point to Your Backend

Edit `frontend/.env.production`:
```

REACT_APP_API_URL=https://aria-backend.onrender.com/aria
```

Then rebuild and redeploy frontend:
```bash
cd frontend
npm run build
npx gh-pages -d dist --no-history
```

## How to Manage Your Data

### View / Query the Database

From Render dashboard → `aria-db` → **Connect** → copy the psql command:
```bash
psql postgresql://user:pass@host/aria_db
```

Useful queries:
```sql
-- See all students
SELECT id, full_name, grade, language FROM student;

-- See all users (passwords are hashed - not readable)
SELECT id, username, role, email FROM aria_user;

-- See all sessions
SELECT s.session_code, st.full_name, s.subject, s.status
FROM learning_session s JOIN student st ON s.student_id = st.id;

-- Delete a student and all their data
DELETE FROM student WHERE id = ?;
```

### Take a Database Backup

From Render dashboard → `aria-db` → **Backups** → **Create Manual Backup**

Or via command line:
```bash
pg_dump postgresql://user:pass@host/aria_db > aria_backup_2026.sql
```

### Restore from Backup
```bash
psql postgresql://user:pass@host/aria_db < aria_backup_2026.sql
```

## Your Responsibilities as Data Owner

As the app owner you are legally responsible for:

### 1. Keep the server running
- Render free tier spins down after 15 min of no traffic (first request takes ~30s)
- For always-on: upgrade to Render Starter ($7/month per service)
- For high availability (school): use Render Standard ($25/month)

### 2. Take regular backups
- Set up weekly manual backups in Render dashboard
- Download and store backup files somewhere safe (Google Drive etc.)

### 3. Protect student data (especially children)
- Never share database credentials with anyone
- Rotate passwords every 6 months
- Only create accounts for real, verified teachers and parents
- Do not share student names/data outside the app

### 4. Respond to data deletion requests
- If a parent asks "delete my child's data" → you must do it
- Use the psql queries above to delete by student ID
- This is a legal right in most countries (GDPR Right to Erasure)

### 5. Secure your Render account
- Enable 2-Factor Authentication on Render.com
- Never share your Render login
- Enable 2FA on GitHub too

## Security Architecture (End-to-End)

```
User's Phone/Browser (HTTPS)
        │
        ▼
GitHub Pages CDN  ──── Static React files (no data stored here)
        │
        │ HTTPS API call with JWT token
        ▼
Render.com (Singapore)
  Spring Boot API
  ├─ Validates JWT token on every request
  ├─ Checks role (ADMIN / TEACHER / PARENT)
  ├─ Filters data by role (parent sees only own children)
  └─ Writes/reads from database
        │
        ▼
PostgreSQL Database (Render — private network)
  ├─ Not accessible from internet
  ├─ Only accessible from Spring Boot (same Render network)
  ├─ Passwords stored as BCrypt hash (unreadable)
  └─ All PII encrypted at rest by Render
```
